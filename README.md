# Movie Ticket Booking System

This project is a backend-intensive movie ticket booking system built with the goal of mastering real-world backend engineering challenges.

Instead of focusing on UI, this system simulates production-level scenarios where failures, concurrency issues, and edge cases must be handled gracefully.

The primary objective is to evolve from writing basic CRUD APIs to building a system that is robust, scalable, and fault-tolerant.

---

## Major Focus

1. Designing a real-world booking flow  
2. Handling high concurrency seat booking  
3. Preventing race conditions & double booking  
4. Managing transactions & rollbacks  
5. Implementing authentication & role-based access  
6. Building secure and reliable APIs  
7. Simulating failure scenarios (timeouts, payment failures)  
8. Writing clean, maintainable backend code  

---

## Real-World Problems Being Solved

- What happens when two users try to book the same seat?  
- How to handle payment success but booking failure?  
- How to ensure data consistency under heavy load?  
- How to design retry mechanisms and idempotent APIs?  
- How to recover from system crashes during booking?  

---

## Core Features

- Movie & show management  
- Seat selection & locking system  
- User authentication & authorization
